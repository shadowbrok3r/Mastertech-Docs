# Getting Started

Welcome to Mastertech — the all-in-one technician tool for PCL stores. This page will walk you through account setup, logging in for the first time, and the fundamental workflow every tech uses every day.

> See [[0. Table of Contents]] for a full list of documentation pages.

---

## Download

Download the latest `Mastertech.exe` from the website:
**[master-tech.app](https://master-tech.app)**

It is a portable executable — no installation needed. Keep it on your tech flash drive and/or pin it to your taskbar on your tech machine.

You can also download the latest version at any time from the [[Downloads]] tab inside the app itself.

---

## Creating Your Account

Before using Mastertech you will need to create an account at [master-tech.app](https://master-tech.app).

- Use your **PC Laptops email address** to register.
- Set your **store** and **initials** carefully — these are difficult to change after the fact.
- Your account is shared between `Mastertech.exe` and the MtechServer web app.

> ⚠️ There is currently no self-serve password reset. If you lose your password or made a mistake during setup, contact Logan.

---

## Signing In

Launch `Mastertech.exe` and sign in with your credentials. The app will connect to MtechServer and load your personal task board alongside the default layout.

Your login is also used to identify you as the technician or salesperson on [[3. TUR Sheet|TUR Sheets]], associate tasks, and track sales in [[Sales Tracker]].

---

## The Core Workflow

The central loop for every technician looks like this:

```
Check-in shelf  →  Run Mastertech  →  Submit TUR Sheet  →  Task appears on web  →  Salesperson calls customer
```

1. **Pick up a computer** from the check-in shelf.
2. **Open Mastertech.exe** on the customer's machine (or your tech machine).
3. **Pull the ticket** — enter the Everest service number or Prestashop order number in [[3. TUR Sheet|TUR Sheet]] to auto-fill customer info and check-in notes.
4. **Fill out recommendations** — document hardware findings, upgrade options, and service notes.
5. **Submit the TUR** — this creates a [[My Tasks|Task]] assigned to the appropriate salesperson on the web app.
6. **Salesperson calls** the customer using the ammo from the task.
7. If there's a sale or follow-up repair, the task gets **reassigned back to you**.

> 💡 Always check **Send System Info** before submitting — this attaches hardware data to the task so salespeople and future techs have full context.

## The Interface

Mastertech uses a dockable tab system. Every "View" is a tab that can be:
- **Dragged** to a new position
- **Stacked** with other tabs (like browser tabs)
- **Split** into side-by-side panels
- **Opened/closed** from the **View** menu at the top left

See [[2. Views and Layout]] for a full explanation of the docking system.

### Default Layout

When you first open Mastertech, the default layout includes:
- **[[3. TUR Sheet|TUR Sheet]]** — center stage
- **[[File Browser]]** — docked to the left
- Several utility tabs underneath

## Key Concepts

| Concept | What it means |
| ------- | ------------- |
| **TUR Sheet** | The form you fill out to send computer info to a salesperson. Creates a Task. |
| **Task** | A work item on the web app representing a customer's computer or service. |
| **Ammo** | The hardware info, recommendations, and notes a salesperson uses to call a customer. |
| **Initials** | Your `first.last` email prefix (e.g. `brett.knowlton`) — used everywhere for attribution. |
| **Prestashop** | The POS system. Order numbers are used to pull customer data. |
| **Everest** | Legacy service order system. Service numbers are also supported for ticket lookups. |
