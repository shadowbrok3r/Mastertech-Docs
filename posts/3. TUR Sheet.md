# TUR Sheet

The TUR Sheet is the most important tab in Mastertech for technicians. It's how you send hardware information, recommendations, and customer data to your salesperson — creating a **Task** on the web app that the salesperson uses to call the customer.

> See [[1. Getting Started]] for the full workflow context, and [[4. Task Boards]] for what happens after you submit.

---

## Pulling a Ticket

Before filling out the TUR Sheet you should pull the existing ticket to auto-populate customer data.

### Service # Field

The **Service #** field at the top accepts either an **Everest** service order number or a **Prestashop** order number.

#### From Everest
Type the service order number and click **Get Ticket**. Mastertech will pull:
- Customer name
- Phone numbers
- Check-in notes
- Salesperson and/or technician initials

#### From Prestashop
Type the Prestashop order number **excluding any letters and leading zeros**, then click **Get Prestashop**. The same customer fields will be populated from the Prestashop order.

> 💡 Clicking **Get Keys** after entering the service number will also pull the **Webroot** and **SuperAntiSpyware** keys directly from Scaffold. Click the key buttons after pulling to **copy them to your clipboard**.

---

## Salesman / Technician Fields

After pulling a ticket, the salesman and technician fields populate in the format:

```
first_name.last_name
```

For example: `brett.knowlton`

This is derived from the employee's PC Laptops email (without the domain). It's used to identify employees consistently across Everest, Prestashop, and the Mastertech system.

> ⚠️ Make sure to verify (and update if needed) these fields — especially if the auto-filled salesperson is on their day off.

## System TUR Info

This section is to the right of the ticket-pulling section and documents the hardware test results and metadata for the TUR.

### Hardware Test Results

Use the dropdown selectors to record results for:
- **RAM** — pass/fail/suspect
- **SSD** — pass/fail/suspect
- **HDD** — pass/fail/suspect/not tested/etc.

### Date Picker

The calendar button in the top right of this section lets you set the **due date** for the task. Default is today, but you can assign it to tomorrow or further out if the customer isn't available for a call right away.

### Send System Info

The **Send System Info** checkbox attaches a snapshot of the machine's hardware to the task. This includes CPU, RAM, GPU, storage, and other system details pulled from the machine Mastertech is running on.

> ✅ **Always check this box** unless Mastertech is running on a different machine than the customer's — for example, if you're submitting a TUR for a dead motherboard and running Mastertech on your own tech machine.

---

## Recommendations Section

This is where you write your technician notes and sales recommendations. Be thorough — this is the ammo the salesperson uses on the call.

Include:
- What's wrong with the machine
- What you recommend (upgrades, replacements, new system)
- Any relevant observations (battery health, age, specs)
- Whether a data transfer is needed
- SEB status (see below)

---

## SEB / Carbonite

If the customer has **SEB** (SecureAnywhere Backup) or **Carbonite**, this section lets you:
- Check SEB activation status (auto-checks if SEB is detected when Mastertech opens — check the Logs tab for output)
- Record the subscription info for the salesperson

> *Full SEB check via button is still in development. Auto-detection runs on startup.*

---

## Submitting the TUR

Once all sections are filled out, click **Submit TUR** to send everything to the web app. This will:

1. Create a new **Task** visible in [[Store Tasks]] (and [[My Tasks]] for the assigned salesperson)
2. Attach all hardware info, customer data, and recommendations
3. Notify the relevant salesperson

After submission you'll see a confirmation toast. The task is now live on the web.

---

## Part Order Integration

If a part needs to be ordered, switch to the [[Part Order]] tab — it will inherit the customer and ticket fields already filled in from the TUR Sheet, saving you from re-entering information.
