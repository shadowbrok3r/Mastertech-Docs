# Logs

The Logs tab is the **in-app runtime log viewer** for Mastertech. It shows a live stream of log messages generated by the application — useful for diagnosing errors, verifying that scripts ran correctly, and seeing what's happening behind the scenes.

> Shared tab — available in both `Mastertech.exe` and at [master-tech.app](https://master-tech.app). See [[0. Table of Contents]].

---

## What It Shows

Log messages from all parts of the application, color-coded by severity:

| Level | Color | When it appears |
| ----- | ----- | --------------- |
| **INFO** | White/gray | Normal operation messages |
| **DEBUG** | Dim | Detailed diagnostics (usually filtered out) |
| **WARN** | Yellow | Something unexpected but non-fatal |
| **ERROR** | Red | Something failed — worth investigating |

Common things you'll see logged:
- Database connection status
- Script start/completion/failure
- WebSocket connection events
- TUR submission results
- SEB/Webroot check results
- File transfer progress
- API call responses

---

## Controls

- **Clear** — Clear the current log buffer
- **Auto-scroll** — Keep the view scrolled to the latest message
- **Filter** — Filter by log level or keyword (if available)

---

## When to Use This

- **After a script runs** — Verify it completed successfully or see why it failed
- **Debugging TUR issues** — Check what happened when Get Ticket or Submit TUR was clicked
- **Checking SEB status** — SEB auto-check results appear here on startup
- **WebSocket issues** — Connection/disconnect events are logged
- **Before filing a [[Bug Tracker|bug report]]** — Copy the relevant log lines into your bug description

---

## Sharing Logs

If you need to report an issue to Logan or submit a [[Bug Tracker|bug report]], copy the relevant log output from this tab and include it in the report. The log often contains the exact error message that explains what went wrong.
