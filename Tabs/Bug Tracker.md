# Bug Tracker

The Bug Tracker tab lets you submit bug reports for Mastertech directly from within the app. Reports go straight to the Mastertech GitHub issue tracker — no need to navigate to GitHub manually.

> Available in both the desktop app (as "Bug Tracker") and on the web (as "BugReport"). See [[0. Table of Contents]].

---

## How to Submit a Bug Report

1. **Title** — Write a short, descriptive title for the issue (e.g., "TUR Sheet crashes when pulling Prestashop order with long name")
2. **Description** — Describe what happened, what you expected to happen, and steps to reproduce it
3. **Submit** — Click the submit button to create the issue on GitHub

Mastertech will automatically include relevant version and environment info alongside your report.

---

## Tips for Good Bug Reports

- **Be specific** — "The app crashed" is less useful than "Clicking Get Ticket with a blank service number field crashed the app"
- **Include steps** — What exactly were you doing when it happened?
- **Note the version** — Check the [[Downloads]] tab to see what version you're on
- **Check the [[Logs]] tab** — Copy any relevant log output into the description
